Documents /v1/documents
Submit PDFs, query pipeline status, retrieve extracted fields, and apply human corrections.
{% call endpoint_block("GET", "/v1/documents/", "Search & list documents") %}
Query parameters
NameTypeRequiredDescription
statusstringoptionalFilter by pipeline status (e.g. COMPLETED, NEEDS_REVIEW, FAILED)
document_classstringoptionalFilter by document class ID
shipment_idstringoptionalFilter documents linked to a shipment
field_namestringoptionalFilter by extracted field name
field_valuestringoptionalFilter by field value — append * for prefix wildcard
date_from / date_tostringoptionalISO 8601 date range on created_at
page / page_sizeintegeroptionalPagination — default 1 / 20, max page_size 100
Response
{ "total": integer, "page": integer, "page_size": integer, "pages": integer, "documents": [DocumentOut] }
{% endcall %} {% call endpoint_block("POST", "/v1/documents/submit", "Submit a PDF for processing") %}
Body — multipart/form-data
NameTypeRequiredDescription
filefilerequiredPDF file, max 50 MB
document_classstringoptionalOverride AI classification with a known class ID
variant_keystringoptionalForce a specific variant instead of auto-discovery
prioritystringoptionalstandard (default) or express
webhook_urlstringoptionalPer-document callback — overrides global webhook subscriptions
metadatastringoptionalJSON string of arbitrary key-value pairs stored on the document
skip_stagesstringoptionalComma-separated stage names to skip: MATCHING, POSTING
Response — 200 DocumentOut
{ "id": "uuid", "status": "RECEIVED", "file_name": string, "document_class": string | null, "priority": "standard" | "express", "created_at": ISO8601 }
409 Conflict — returned if the file's SHA-256 matches an existing document. The response body contains the existing document's ID.
